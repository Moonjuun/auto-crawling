# Git 커밋 규칙

## 커밋 메시지 형식
```
[태그] 한국어 설명
```

## 태그 종류
| 태그 | 상황 |
|---|---|
| `[Feat]` | 새로운 기능 추가 |
| `[Fix]` | 버그 수정 |
| `[Design]` | CSS·UI·디자인 변경 |
| `[Refactor]` | 기능 변경 없이 코드 구조 개선 |
| `[Chore]` | 설정 파일·패키지 설치 등 |
| `[Docs]` | 문서 수정 |

## 커밋 프로세스
"커밋해줘" 요청 시 아래 순서를 따르라:
1. `git status`로 변경 사항 확인
2. 위 규칙에 맞는 커밋 메시지 작성
3. `git add . && git commit -m "[태그] 메시지"`
